Cite key format

Syntax of the cite key format

The format for the cite key is a string containing special format specifiers, which are replaced by strings determined by the bibliography data. Most format specifiers have the form of a %-character followed by a single character. It can be followed by optional digits, which serve as parameters for the format specifier. Possible format specifiers are given in the table below.

SpecifierReplaced content description
%a[00]Author last names, with (optional) maximal number of authors and number of characters
%A[0]Author last names with initials, with (optional) maximal number of authors
%t[0]Title, with (optional) maximal length
%YYear without century
%yYear with century
%mMonth as decimal number
%r[1]Random lowercase letter, with (optional) number of characters
%R[1]Random uppercase letter, with (optional) number of characters
%d[1]Random number, with (optional) number of digits
%u[1]Unique lowercase letter, with (optional) number of characters (only at the end)
%U[1]Unique uppercase letter, with (optional) number of characters (only at the end)
%n[1]Unique lowercase letter, with (optional) number of characters (only at the end)
%0Escaped digit (0-9)
%{Field}[0]Arbitrary field, with (optional) maximal length

In the table above, the numbers between square brackets after the specifiers indicate the optional parameters (the brackets themselves are not part of the format, the first parameter after an author specifier should be a single digit). Indicated in the table are the parameter values that are used when they are not entered.

The special value of zero for an optional parameter is interpreted either as all (for the ‘author’, ‘title’, or ‘field’ specifiers), sufficient (for the ‘unique’ specifiers, i.e., sufficient to create a unique cite key), or none (for the ‘random’ specifiers).

Apart from the specifiers, the format can only contain unaccented alphanumeric characters (a-zA-Z0-9), (semi-)colons (:;), dashes (-) or periods (.). Any generated part of the cite key will be converted to contain only those characters. For example, accents are removed and spaces will be converted to dashes. To use explicit digits right after a format specifier, use one of the specifiers %0-%9.

The ‘unique’ specifiers %u, %U and %n can only be used at the end of the format string.

Examples

Bibliography:
M. McCracken, A. Maxwell, J. Howison, M. Routley, S. Spiegel, S. S. Porst, and C. M. Hofman,
Bibdesk, a great application to manage your bibliographies,
SourceForge, 1:96, 11 2004.

Format string:%a1:%Y%r2
Resulting cite key:McCracken:2004yc

Format string:%a1:%Y%u
Resulting cite key:McCracken:2004b

Format string:%a03%y%m
Resulting cite key:McCMaxHowRouSpiPorHof0411

Format string:%a1%y%t20
Resulting cite key:McCracken2004Bibdesk-a-great-appl

Format string:%a24:%{Journal}%{Volume}
Resulting cite key:McCrMaxw:SourceForge1
